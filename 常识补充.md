```
{
    struct timeval tv;
    gettimeofday(&tv,NULL);
    return tv.tv_sec*1000000+tv.tv_usec;
}
```

#### 步骤：

1.先通过性能测试的方式找到“性能瓶颈”

2.找到性能瓶颈，对症下药

​	a)某个计算量太大，占用很多的CPU资源

​		可以使用多线程的方式，也可以使用缓存（将常用的计算结果保存）的方式（二八原则）

​	b)某个操作IO成为瓶颈了

​		可以使用多线程的方式，也可以考虑使用缓存的方式，也可以使用IO多路复用的方式，

​		使用缓冲区的方式

​	c)CPU和IO都成为瓶颈了
